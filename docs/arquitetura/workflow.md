# Workflow de Desenvolvimento

Este documento define o fluxo oficial de desenvolvimento do Smart Exit School.

---

## Etapas

1. Definir regras de negócio.

↓

2. Registrar decisões arquiteturais (ADR), quando necessário.

↓

3. Modelar o banco de dados.

↓

4. Criar Migration.

↓

5. Atualizar documentação.

↓

6. Implementar Services.

↓

7. Implementar Frontend.

↓

8. Testar.

↓

9. Commit.

↓

10. Push.

---

# Regras

- Nunca alterar migrations publicadas.
- Sempre criar novas migrations.
- Nunca implementar funcionalidades sem modelagem prévia.
- Toda alteração estrutural deve possuir documentação.
- Toda decisão arquitetural relevante deve ser registrada nas ADRs.
